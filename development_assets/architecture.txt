2. REQUEST FLOW (Session Auth)
--------------------------------
  1. User submits login form → POST /users/login/
  2. Django authenticates credentials (email + password)
  3. Session created → session ID stored in browser cookie
  4. User redirected to /users/dashboard/
  5. Dashboard view checks user.role → renders role-specific template
  6. All subsequent requests carry session cookie for auth

---

3. REQUEST FLOW (REST API)
----------------------------
  1. Client sends POST /api/auth/login/ with JSON {email, password}
  2. Django validates → returns session cookie (or token if added)
  3. Client uses session cookie for subsequent API calls
  4. API returns JSON responses (status, data, errors)

---

4. ROLE-BASED ACCESS CONTROL
------------------------------
  Role       | Access Level
  -----------|--------------------------------------------------
  admin      | All apps: users, inventory, donations, requests, fraud
  donor      | donors, donations (own), profile
  hospital   | hospitals, requests (own), profile

  Enforcement: @login_required + role_required decorator on all views
  Admin panel: /admin/ (Django built-in, superuser only)

6. API ENDPOINT MAP
---------------------
  Method  | Endpoint                      | Description
  --------|-------------------------------|----------------------------------
  POST    | /api/auth/register/           | Register new user
  POST    | /api/auth/login/              | Login, returns session
  POST    | /api/auth/logout/             | Logout, clears session
  GET     | /api/auth/me/                 | Get current user info
  GET     | /api/inventory/               | List blood inventory
  POST    | /api/inventory/               | Add blood unit (admin)
  GET     | /api/requests/                | List blood requests
  POST    | /api/requests/                | Create blood request (hospital)
  PATCH   | /api/requests/<id>/           | Update request status (admin)
  GET     | /api/donations/slots/         | List donation slots
  POST    | /api/donations/book/<id>/     | Book a slot (donor)
  GET     | /api/fraud/logs/              | List fraud flags (admin)
  PATCH   | /api/fraud/logs/<id>/resolve/ | Resolve a fraud flag (admin)
