Requirement Analysis & Planning (Weeks 1–2)
Finalize requirements as user stories (e.g., donor registration with verification).
Design an ER diagram covering users, donations, requests, inventory, blood units.
Database schema (sqlite): users (role-based), donors, hospitals, inventory, donation history, requests, fraud logs.
Set up tools: GitHub, VS Code, MySQL Workbench, Postman.
Deliverables: requirements doc, wireframes, database schema.

System Design & Setup (Weeks 3–4)
Architecture: Client → Frontend → Backend APIs → sqlite.
Project structure: frontend, backend, docs folders.
Backend setup (Flask/FastAPI), authentication (JWT/session + bcrypt).
Basic frontend: login, dashboard skeletons per role.
Milestone: skeleton app with login, role redirection, API endpoints tested.

Core Development Phase I (Weeks 5–7)
Admin panel: user management, inventory, reports.
Donor panel: registration, profile/history, book donation slot.
Hospital panel: registration, request blood, view status.
Inventory: add/update blood units, expiry alerts.
Security: input validation, role checks, audit logs.
Milestone: end-to-end flows working.

Core Development Phase II + Fraud Detection (Weeks 8–9)
Real-time inventory (refresh/polling).
Fraud detection: rule-based (duplicate checks, fuzzy matching), anomaly flags, optional ML (Isolation Forest/clustering).
Admin alerts, suspicious activity logs.
History views, notifications.
Milestone: fraud rules, alerts, CRUD flows stable.

Testing (Week 10)
Unit tests (backend APIs), manual testing (edge cases).
Security: SQL injection, XSS, auth bypass.
Usability: feedback from users.
Bug fixes, query optimization.
Milestone: stable, bug-minimal version.

Deployment & Polish (Week 11)
Deploy backend/frontend (Render.com/Heroku, MySQL hosted).
Domain setup, HTTPS, documentation, demo materials.
Milestone: live/demo-ready system.

Maintenance, Documentation & Submission (Week 12)
Final refinements, report/thesis, presentation, backup.
Milestone: project submitted.
